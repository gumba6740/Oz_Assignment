반려동물 호텔 예약시스템 DB 설계

예약 단위 구조 : 1예약 =  1펫 = 1룸 = n서비스


고객
 - 고객id 
 - 이름  
 - 생년월일  
 - 전화번호  
 - 이메일  # UNIQUE
 - 주소 
 - 가입일


펫
 - 펫id
 - 고객아이디 # 참조
 - 종  
 - 품종
 - 생년  
 - 몸무게
 - 비고  # TEXT

객실
 - 룸아이디
 - 방번호  # UNIQUE
 - 타입  
 - 하루 숙박 가격  


예약
 - 예약 아이디
 - 펫id  # 참조
 - 룸id  # 참조
 - 체크인
 - 체크아웃
 - 숙박결제가격 # 숙박일 * 일일 가격
 - 상태  # 예약신청, 예약완료, 투숙 중, 퇴실완료, 퇴실 지연, 취소, 노쇼



서비스 메뉴
 - 서비스아이디
 - 서비스명
 - 서비스 설명
 - 회당 가격



# 서비스의 가짓수를 여러개 정할 수 있기 때문에 예약 테이블과 분리해서 관리
예약_서비스
 - 예약 아이디 # 참조
 - 서비스 아이디
 - 횟수
 - 가격 # 가격고정을 위해 기록


# 고객아이디를 확인하려면 3단계를 거쳐야하기 때문에 고객아이디를 넣음
결제
 - id
 - 고객아이디
 - 예약 아이디
 - 결제일
 - 총 결제금액  # 예약테이블의 숙박결제금액 + 
                         예약_서비스 테이블의 횟수 * 가격
 - 결제수단
 - 상태 # 대기, 완료, 취소, 실패, 환불
